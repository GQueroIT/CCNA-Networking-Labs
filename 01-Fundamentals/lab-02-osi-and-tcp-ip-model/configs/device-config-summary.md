# Device Configuration Summary

## Overview

This lab did not require direct CLI configuration. All setup was completed using basic settings within Cisco Packet Tracer.

Instead of focusing on device configuration, the primary goal was to observe how data flows through the network and how protocols interact across different layers.

---

## Environment Setup

The following configurations were applied using the Packet Tracer interface:

- PCs were assigned static IP addresses
- Server had HTTP services enabled

No manual CLI configuration was performed on networking devices.

---

## Protocol Behavior Observed

During simulation, the following protocol interactions were analyzed:

- ARP resolving MAC addresses before communication begins
- TCP establishing a connection using the three-way handshake
- HTTP operating at the application layer and relying on lower-layer protocols

---

## Key Focus of the Lab

This lab emphasized understanding network communication rather than configuring devices.

Using Simulation Mode allowed for step-by-step visibility into how packets move through the OSI and TCP/IP models.

---

## Takeaway

Even without CLI configuration, this lab was critical in building a foundational understanding of how network communication works.

Not every lab focuses on command-line work — understanding protocol behavior is equally important for troubleshooting and real-world networking.
